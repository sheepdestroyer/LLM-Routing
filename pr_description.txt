🎯 **What:** The `get_goose_sessions` function in `router/main.py`, which is responsible for querying the live mounted SQLite goose database to fetch the latest agentic sessions, was previously untested. This PR adds dedicated unit tests for it.

📊 **Coverage:**
* Happy path: DB exists, query succeeds, and returns correctly structured and sorted list of sessions.
* DB Not Found path: Handled gracefully and returns an empty list.
* DB Error path: Simulation of a locked database resulting in `sqlite3.OperationalError`, caught and returns an empty list.

✨ **Result:** Increased codebase reliability by ensuring the SQLite session fetching implementation works as expected, catches potential DB connectivity and filesystem errors gracefully, and remains regression-free.
