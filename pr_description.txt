🎯 **What:** Hardcoded security secrets (`NEXTAUTH_SECRET`, `SALT`, and `ENCRYPTION_KEY`) in `pod.yaml` were replaced with placeholders. The `start-stack.sh` deployment script was updated to dynamically generate these cryptographically secure values (using `openssl rand`) and store them in `.env`.

⚠️ **Risk:** Hardcoded secrets in source control (like `pod.yaml`) can lead to unauthorized access, privilege escalation, or data decryption if the repository is compromised or accessed by unauthorized individuals.

🛡️ **Solution:** Implemented environment-based secret injection. The deployment script now ensures these keys are automatically and securely generated on first deployment if they are missing, appended to `.env` for persistence, and dynamically injected into the pod configuration at runtime, keeping them safely out of source control.
