🎯 **What:** Removed the hardcoded PostgreSQL password (`postgres-local-pw-2026`) from `pod.yaml` and replaced it with a dynamic environment variable insertion (`postgres-password-***`) that `start-stack.sh` replaces at runtime. Also addressed two CI security warnings in `get_pr_status.py` regarding command injection vulnerabilities from `subprocess.run()`.
⚠️ **Risk:** Storing credentials in plain text in configuration files (like `pod.yaml`) can lead to unauthorized access to the database or unauthorized operations if the codebase is exposed or shared. It creates a critical security risk. The `subprocess.run()` warnings identified unsafe shell execution behavior.
🛡️ **Solution:** Updated `pod.yaml` to use a generic placeholder instead of the password. Modified `start-stack.sh` to load or securely generate (`openssl rand -hex 16`) a password from the `.env` file, and securely inject it at runtime into the `pod.yaml` file. Modified `get_pr_status.py` to use `shlex.split()` with a static array of arguments and set `shell=False` to securely execute shell commands.
